## 1. Memory Search 架构

Memory Search 和对话模型是**完全独立的两个组件**：

| 组件 | 当前模型 | 作用 |
|------|----------|------|
| 对话模型 | `openrouter/stepfun/step-3.5-flash:free` | 理解和生成回答 |
| Embedding 模型 | `Qwen text-embedding-v3` | 把文字转成向量供检索 |

**关键点：** 两者可以任意组合，互不影响。

---

## 2. Memory Search 正确配置路径

必须写在 `agents.defaults.memorySearch` 下（不是顶层）：

```json
{
  "remote": {
    "baseUrl": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "apiKey": "Qwen API Key"
  },
  "extraPaths": ["/home/xzy0626/.openclaw/workspace"]
}
```

**extraPaths** 使 workspace 根目录文件也被索引。

## 5. 知识投喂方式

| 方式 | 用法 |
|------|------|
| **粘贴内容** | 直接告诉龙虾"存入记忆库，主题是XXX：[内容]" |
| **飞书文档** | 说"把这个飞书文档收录到知识库：[链接]" |
| **永久规则** | 让龙虾写进 AGENTS.md（每次对话必读） |

**存储位置：** `~/.openclaw/workspace/memory/YYYY-MM-DD-主题.md`
